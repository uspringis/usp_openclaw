## MyFitness Calendar Sync Past Event Retention

Ugis noticed MyFitness calendar entries were removed after time passed and he had attended them. He only wants removal when a booking is canceled.

Investigation:
- OpenClaw cron job `MyFitness Calendar Sync` (`bf111852-60da-438a-af4b-68ae9f760aa7`) runs hourly at minute 7 in `Europe/Riga`.
- It executes `/home/uspringis/myfitness-assistant/run_calendar_sync.sh`, which runs `sync_myfitness_calendar.py`.
- Root cause: the sync fetched bookings with `include_past=False` but listed calendar events from yesterday onward. Once an attended class disappeared from MyFitness bookings, the previous-day calendar event was considered stale and deleted.

Fix:
- Updated `sync_myfitness_calendar.py` so stale `MyFitness:` events are deleted only if their start time is still in the future.
- Past stale `MyFitness:` events are retained as attended history.
- Updated the MyFitness README to document this behavior.

Verification:
- `python3 -m py_compile sync_myfitness_calendar.py` passed.
- `uv run sync_myfitness_calendar.py --dry-run` showed no deletions and retained a past `MyFitness: Functional Training` event as unchanged.
