assistant: Found it.

`/model openai/gpt-5.5 --all` fails because `/model` does **not** support `--all` in OpenClaw docs.  
So parser treats the whole tail as model text (`"openai/gpt-5.5 --all"`), then rejects it as “model not allowed”.

What to use instead:

1. Current session only:
`/model openai/gpt-5.5`

2. Set global default model (all new sessions):
`openclaw models set openai/gpt-5.5`

3. If you still get “model is not allowed”, add it to allowlist:
`openclaw config set agents.defaults.models '{"openai/gpt-5.5":{}}' --strict-json --merge`

4. Verify availability:
`openclaw models list --provider openai`
